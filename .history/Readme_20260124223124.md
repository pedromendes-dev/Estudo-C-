# Iniciando Estudos em CSharp

O comando `dotnet new console` serve para criar um novo projeto de aplicação de console em C# usando o .NET SDK.

Se você quiser dar um nome específico ao projeto, pode usar:

```bash
dotnet new console -n MeuProjeto
```

### Compilar e executar

- Compilar com o compilador C# (csc):

```powershell
csc Teste.cs
```

- Executar no Windows (PowerShell):

```powershell
.\Teste.exe
```

- Compilar gerando um executável com nome específico sem deixar resíduos na pasta:

```powershell
csc -out:output.exe Teste.cs
.\output.exe
```

- Para projetos completos com `Program.cs`, `dotnet run` é a opção mais recomendada:

```bash
dotnet run
```

### Sintaxe e convenções

#### Como é composto um projeto, classes e suas convenções

`class` é um molde para criar objetos; ela é um container de métodos e propriedades. Métodos dentro da classe só rodam se forem chamados.

O método `Main` é especial: ele roda automaticamente quando você executa `dotnet run`.

- Ele precisa ser `static` porque o runtime (CLR) chama esse método diretamente pela classe, sem criar um objeto antes.

---

#### Modificadores de acesso (public, private, protected, etc.)

- Eles servem para controlar quem pode acessar atributos e métodos de uma classe.

1. **public**: Visível para todos; qualquer classe pode acessar. Usado quando você quer que o método ou atributo seja aberto.

```csharp
public class Pessoa {
    public string Nome; // qualquer um pode acessar
}
```

1. **private**: Visível apenas dentro da própria classe. Usado para encapsular dados, protegendo-os de acesso externo.

```csharp
public class Pessoa {
    private int idade; // só a própria classe pode acessar

    public void SetIdade(int valor) {
        idade = valor; // manipulado por método público
    }
}
```

1. **protected**: Visível dentro da classe e em subclasses (herança). Usado quando você quer que apenas quem herda a classe tenha acesso.

```csharp
public class Animal {
    protected void Respirar() {
        Console.WriteLine("Respirando...");
    }
}

public class Cachorro : Animal {
    public void Latir() {
        Respirar(); // pode acessar porque herdou
        Console.WriteLine("Au au!");
    }
}
```

---

#### Escopo e chamada de métodos conforme o modificador de acesso

- **public**: pode ser chamado de qualquer lugar
- **private**: só pode ser chamado dentro da mesma classe
- **protected**: só dentro da classe e classes derivadas
- **internal**: só dentro do mesmo assembly/projeto
- **static**: chamado pela classe, não precisa de instância
- **abstract**: método sem implementação que obriga classes derivadas a implementarem

Observação: uma instância é um objeto criado a partir de uma classe (instanciar = criar um objeto a partir de uma classe). Um método `static` não está ligado a nenhum objeto específico, mas sim à própria classe.

Construtor é um método especial que é chamado automaticamente quando você instancia (cria) uma classe. Ele serve para inicializar o objeto.
